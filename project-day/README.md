## วิธีทำงาน

ในไฟล์ของตัวเองจะมี **คำอธิบายขั้นตอน** อยู่ในฟังก์ชันแล้ว เขียนโค้ดแทนที่บรรทัด `pass`

ทดสอบไฟล์ตัวเองได้ทันทีโดยไม่ต้องรอเพื่อน:

```bash
python -m personnel.add_member      # (เปลี่ยนเป็นโฟลเดอร์.ชื่อไฟล์ของตัวเอง ไม่ต้องมี .py)
```

ทดสอบทั้งระบบรวมกัน:

```bash
python main.py
```

- ขึ้น `!! ... ยังไม่ถูกเขียน` = ไฟล์ฟังก์ชันนั้นยังไม่เสร็จ ไปตามเพื่อนได้เลย 😄
- ขึ้น `!! เมนูนี้ยังไม่ถูกเชื่อม` = หัวหน้ายังเขียนเมนูนั้นใน `main.py` ไม่เสร็จ ไปตามหัวหน้า 😆

## ของใหม่ 2 อย่างที่ยังไม่ได้เรียนในคลาส (ไม่ต้องกลัว)

- **`from personnel.add_member import add_member`** = การดึงฟังก์ชันจากไฟล์อื่นมาใช้ (นี่แหละคือเหตุผลที่แยกไฟล์กันแล้วงานเพื่อนมารวมกับเราได้!) — เขียนให้หมดแล้ว ไม่ต้องเขียนเอง
- **`None`** = ค่าพิเศษที่แปลว่า "ว่างเปล่า/ไม่มี" ใช้เป็นสัญญาณตอบกลับ เช่น ค้นหาแล้วไม่เจอ

## ขั้นตอน GitHub

### 🧑‍✈️ หัวหน้ากลุ่ม (ทำครั้งเดียวตอนเริ่ม)

1. กดปุ่ม **Use this template → Create a new repository** ตั้งชื่อ repo ของกลุ่ม
2. ไปที่ **Settings → Collaborators → Add people** แล้วเชิญเพื่อนทุกคนในกลุ่ม
3. บอกลิงก์ repo ให้เพื่อน (เพื่อนต้องกดรับคำเชิญในอีเมล/หน้า GitHub ก่อน)

### 🧑‍💻 สมาชิกทุกคน (รวมหัวหน้า)

```bash
# 1. โคลนโปรเจกต์ลงเครื่อง (ทำครั้งเดียว)
git clone <ลิงก์ repo ของกลุ่ม>
cd <ชื่อโฟลเดอร์>

# 2. เขียนโค้ดในไฟล์ของตัวเองเท่านั้น แล้วส่งขึ้น GitHub
git add <ไฟล์ของตัวเอง>
git commit -m "เขียน add_member เสร็จแล้ว"
git pull          # ดึงงานเพื่อนมาก่อนเสมอ
git push          # ส่งงานตัวเองขึ้นไป

# 3. อยากได้งานล่าสุดของเพื่อนมารันรวม
git pull
```

> 💡 commit + push ทุกครั้งที่ฟังก์ชันเริ่มใช้งานได้ อย่ารอทำเสร็จหมดแล้วค่อย push ทีเดียว
